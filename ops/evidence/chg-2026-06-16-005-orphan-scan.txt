ROOT CAUSE of the qa_failed/correctness finding:
  v1 logic flagged an object only when its instance label had NO live Helm
  release secret (release entirely gone). The real chg-2026-06-16-004 orphans
  had a LIVE owning release (prometheus-operator-crds, CRD-only chart) whose
  CURRENT rendered manifest simply no longer contained those SMs/PrometheusRules.
  v1 therefore reported orphan_monitoring_count:0 for the real class.

FIX — detect orphan-of-revision:
  An object is an orphan iff Helm-managed (annotation meta.helm.sh/release-name
  OR label app.kubernetes.io/managed-by=Helm) AND no ownerReferences AND
  EITHER (a) its owning release has no live deployed secret  [release-gone class]
  OR     (b) its Kind/name is NOT present in that release's CURRENT rendered
             manifest                                         [orphan-of-revision].
  Owning release = meta.helm.sh/release-name annotation, falling back to the
  app.kubernetes.io/instance label.
